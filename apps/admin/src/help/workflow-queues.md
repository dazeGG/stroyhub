# Рабочие очереди

Рабочие очереди показывают source products, которые pipeline уже классифицировал
по следующему действию. Все очереди используют одну страницу, но разные правила
и доступные действия.

![Рабочие очереди](/help/screenshots/workflow-queues.png)

## Блоки на скриншоте

1. **Разделы работы**: быстрый переход между очередями.
2. **Текущая очередь**: название очереди и количество карточек.
3. **Переключатель очередей**: вкладки `Можно принять`, `Нужна проверка`,
   `Проблемы данных`, `Похожие товары`, `В каталоге`.
4. **Поиск по карточкам**: фильтрует текущую очередь по названию.
5. **Карточка решения**: source product с магазином, категорией, ценой,
   сигналами, конфликтами и извлеченными атрибутами.
6. **Предлагаемое действие**: решение pipeline и ручные действия оператора.

## Очереди

- **Можно принять**: товары, которые прошли проверки и могут быть созданы как
  canonical products массовым действием.
- **Проверка**: товары, где не хватает уверенности или есть конфликт сигналов.
- **Похожие товары**: source products с candidate matches к существующим
  canonical products.
- **Проблемы данных**: не товары, пустые цены, слишком общие названия и другие
  блокеры качества.
- **В каталоге**: source products, которые уже связаны с canonical product.

## Массовое принятие

![Можно принять](/help/screenshots/workflow-auto-accept.png)

1. **Очередь автопринятия**: показывает только карточки, которые pipeline считает
   безопасными для создания canonical products.
2. **Dry-run проверки**: сначала считает, сколько карточек можно принять и что
   будет пропущено.
3. **Фильтр страницы**: ограничивает batch текущей выборкой.
4. **Навигация по очередям**: позволяет сразу проверить, не появились ли новые
   кандидаты или проблемы данных.

После принятия карточки должна запускаться проверка кандидатов. Если новый
canonical product является candidate match для других source products, эти
товары должны уйти в очередь **Похожие товары**, а не автоприняться без ревью.

## Практическое правило

Не принимай товар вручную, если видишь конфликт по размеру, материалу, сорту,
единице продажи или категории. Такие случаи должны оставаться в ручной проверке.
