# Качество каталога

Главная страница показывает, что сейчас мешает каталогу быть готовым для
публичного продукта. Это точка входа для оператора.

![Качество каталога](/help/screenshots/dashboard.png)

## Блоки на скриншоте

1. **Навигация**: сайдбар со всеми рабочими, каталоговыми, источниковыми и
   техническими разделами.
2. **Назначение экрана**: заголовок и кнопки быстрых действий. Кнопка
   **Обновить** перечитывает сводку, **Открыть очередь** ведет в очередь
   `Можно принять`.
3. **Метрики внимания**: крупные показатели, которые сразу ведут в нужный
   раздел. Здесь видны товары на принятие, проверку, проблемы данных,
   похожие товары и элементы в каталоге.
4. **Очереди оператора**: разложение рабочих задач по queue status. Этот блок
   помогает понять, где продолжать ручную работу.
5. **Здоровье источников**: сжатая сводка по scrape freshness, ошибкам и новым
   source candidates.

## Как пользоваться

Начинай день с этого экрана. Если есть блокеры качества, сначала переходи в
**Проблемы данных** и **Проверка**. Если блокеров нет, можно смотреть
**Похожие товары** и **Нормализованный каталог**.

## Что считается тревожным сигналом

- Растет число проблем данных.
- Есть новые source candidates, но их никто не подтверждает.
- Падает покрытие категориями.
- Есть quality findings уровня blocker.

## Связанные разделы

- **Рабочие очереди**: детальная работа с source products.
- **Загрузки**: если проблема идет от scrape pipeline.
- **Категоризация**: если много товаров без normalized category.
