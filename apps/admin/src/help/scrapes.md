# Загрузки

Раздел **Загрузки** показывает здоровье scraping pipeline: состояние магазинов,
последние scrape runs, скрейпы и агрегированные статусы обработки каталога.

![Загрузки](/help/screenshots/scrapes.png)

## Блоки на скриншоте

1. **Источники**: навигация к магазинам, новым источникам и загрузкам.
2. **Статус загрузок**: заголовок и назначение страницы. Если здесь есть ошибка,
   сначала проверяется admin API и worker.
3. **Фильтры**: сужают страницу по источнику, магазину и статусу scrape run.
4. **Состояние магазина**: показывает scrape metadata магазина: `last_scraped_at`,
   `next_scrape_at`, статус, счетчики ошибок и доступные действия.
5. **Последний запуск**: recent runs с `items_seen`, `items_saved`, статусом и
   текстом ошибки.

## Что проверять первым

- Магазины со статусом `failed` или `partial`.
- Просроченный `next_scrape_at`.
- Резкое падение `items_seen` или `items_saved`.
- Повторяющиеся ошибки одного источника.

## Когда заводить задачу

- Один и тот же магазин стабильно падает.
- Весь источник начал отдавать другую структуру данных.
- `items_seen` стал сильно меньше обычного.
- Scrape runs успешные, но новые source products или price snapshots не появляются.
