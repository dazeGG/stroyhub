# Очередь нормализации

Это технический legacy inbox для source products. Новый рабочий флоу уже
перенесен в разделы **Работа**, но inbox остается полезным для диагностики и
ручной проверки старых сценариев.

![Очередь нормализации](/help/screenshots/normalization-inbox.png)

## Блоки на скриншоте

1. **Техническое**: служебные разделы, которые не являются основным ежедневным
   flow оператора.
2. **Старый inbox**: назначение страницы и текущий режим очереди.
3. **Счетчики**: сколько source products в текущем статусе, в работе и
   заблокировано.
4. **Фильтры**: статус, источник, магазин, категория и поиск.
5. **Карточка источника**: source product, потенциальные матчи и ручные действия.

## Когда использовать

- Нужно сравнить новый workflow с предыдущей очередью.
- Нужно проверить, почему source product не попал в рабочие очереди.
- Нужно вручную воспроизвести старый сценарий нормализации.

## Когда не использовать

Для ежедневной работы лучше идти в **Качество каталога** и **Рабочие очереди**.
Там уже применены новые правила по кандидатам, проблемам данных и автопринятию.
