# Обзор админки

Админка StroyHub нужна для ежедневной проверки качества каталога без терминала.
Она собирает source products, canonical products, источники, очереди решений,
скрейпы, категории и кандидаты связей в один операторский интерфейс.

![Качество каталога](/help/screenshots/dashboard.png)

## Основной цикл оператора

1. Открой **Качество каталога** и посмотри, где есть работа.
2. Разбери очереди в группе **Работа**: сначала блокеры, затем похожие товары и данные.
3. Проверь, что нормализованные товары появились в **Нормализованном каталоге**.
4. Если проблема идет от парсинга или магазина, переходи в **Источники товаров** и **Загрузки**.
5. Если проблема в taxonomy или правилах, используй **Категоризация** и **Категории**.

## Разделы

- **Работа**: текущие очереди оператора и сводка по качеству.
- **Каталог**: нормализованные товары, исходные карточки и качество категоризации.
- **Источники**: магазины, кандидаты новых источников и состояние загрузок.
- **Техническое**: служебные страницы для старого inbox, дерева категорий и сверки связей.

## Как читать скриншоты

Красные прямоугольники выделяют функциональные блоки. Номер на рамке совпадает
с номером в описании под скриншотом. Если блок отвечает за действие, описание
поясняет, какие данные изменяются и какой следующий экран обычно открывать.

## Локальный запуск

```bash
docker compose up -d
```

Админка открывается на `http://127.0.0.1:5173/` и ходит в admin API через
локальный proxy `/api/*`.
